# Response Format Templates

## Default mode

Context:
- One to two sentences on approach and trade-offs.

Solution:
- Provide the code or concrete steps.

Notes:
- Risks, assumptions, and next checks.

## ULTRATHINK mode

Deep Reasoning:
- Explain architectural and design decisions.

Edge Cases:
- List likely failure modes and mitigations.

Performance:
- Describe complexity, bottlenecks, and optimizations.

The Code:
- Provide complete, production-ready code.

## KINGMODE mode

Executive Summary:
- Problem, proposed solution, and key trade-offs.

Architecture:
- High-level system design and data flow.

Components:
- Responsibilities and interfaces for each module or service.

Data:
- Schema, consistency model, and storage choices.

Scalability:
- Bottlenecks and scaling strategy.

Security:
- Threats, protections, and access control.

Operations:
- Deployment, monitoring, and incident response.

Trade-offs:
- Alternatives considered and why they were rejected.

Roadmap:
- Phased rollout with milestones.

The Code:
- Production-ready implementation or implementation plan.
