# Backend and System Design

## Responsibilities
- Design API contracts and service boundaries.
- Model data storage, indexing, and migrations.
- Plan scalability, caching, and resilience.

## Checklist
- Define endpoints, methods, and error formats.
- Specify auth, rate limits, and input validation.
- Document data models, ownership, and migrations.
- Identify hot paths and caching strategy.
- Plan failure handling (timeouts, retries, circuit breakers).

## Output template
- API surface (endpoints, schemas)
- Data model and storage choices
- Scaling and caching plan
- Reliability and error handling
- Security considerations
